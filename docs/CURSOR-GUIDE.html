02 — LLM Patterns baked into the AI

These 9 patterns live in .cursorrules. The AI follows them automatically on every LLM feature — no prompting required.

PATTERN 1 — Always Stream Default behavior
Every completion endpoint uses SSE streaming by default. The AI generates StreamingResponse (FastAPI) or streamed fetch (TS) automatically. Non-streaming only for background jobs.
PATTERN 2 — Structured Output, Never Parse Prose Highest impact
The AI uses response_format=PydanticModel (OpenAI) or tool_use (Anthropic) for ALL extraction. It will actively refuse to write regex-based LLM output parsing and explain why.
PATTERN 3 — Prompt Management as Code Quality multiplier
Prompts live in src/prompts/ as versioned constants (TASK_SYSTEM_V2). No inline f-string soup. Breaking changes get a new version suffix. Old versions kept until traffic migrates.
PATTERN 4 — Retry + Fallback Chain Reliability
Every LLM call wrapped with tenacity exponential backoff. Model chain defined once: gpt-4o → gpt-4o-mini → claude-3-haiku. Falls back automatically on rate limits or errors.
PATTERN 5 — Cost Tracking (Non-Optional) Survival skill
Every LLM call records model, input/output tokens, cost_usd, latency_ms to llm_calls table. Cost constants defined per model. Daily cron alerts you when monthly spend crosses threshold.

This has saved solo founders from $1,000+ surprise API bills.
PATTERN 6 — RAG Pipeline Architecture Core skill
Full RAG spec baked in: semantic chunking (300–800 tokens, 10–15% overlap), pgvector hybrid search SQL (copy-paste ready), retrieve top-20 → rerank to top-5, citations always returned. pgvector + PostgreSQL full-text via RRF — no separate infra needed.
PATTERN 7 — Agent Loops with Hard Caps Safety
Every agent loop gets MAX_AGENT_ITERATIONS = 10 hard cap. Parallel tool execution via asyncio.gather. Structured AgentResult returned — never raises from inside the loop. Every iteration logged with cost.
PATTERN 8 — Langfuse Observability from Day One Debug power
@observe() decorator on every LLM-calling function. Traces latency, tokens, cost, user_id automatically. Free tier is generous. Review weekly — it directly improves product quality.
PATTERN 9 — Rate Limiting on Every LLM Endpoint Protection
20/minute · 100/hour · 500/day per user on all LLM endpoints. Redis-backed. Plan-aware (paid tiers get higher limits). Returns Retry-After header on 429. Generated automatically with every new LLM route.

04 — Hard stops — AI refuses to write this code
In LLM Code
Parsing LLM output with regex or string splits
Infinite agent loops (no MAX_ITERATIONS cap)
User input directly into system prompt content
LLM calls inside a loop without cost check
Ignoring token count on completions
Embedding model mismatch: ingest vs query
Storing embeddings without source metadata
In API / General Code
Stripe webhooks without signature verification
DB calls directly in route handlers
Unauthenticated endpoints (without explicit note)
SELECT * in production queries
Bare except: in Python
any type in TypeScript
Hardcoded secrets or API keys anywhere
